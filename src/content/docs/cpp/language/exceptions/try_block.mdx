### Ordinary `try` block

An ordinary `try` block is a <DocLink dest="cpp/language/statements">statement</DocLink>.

If an exception is thrown from its `compound-statement`, the exception will be matched against the <DocLink dest="cpp/language/catch">handlers</DocLink> in its `handler-seq`:

```cpp
void f()
{
    throw 1;     // NOT handled by the handler below
    try
    {
        throw 2; // handled by the associated handler
    }
    catch (...)
    {
        // handles the exception 2
    }
    throw 3;     // NOT handled by the handler above
}
```

### Function `try` block

A function `try` block is a special kind of <DocLink dest="cpp/language/function#Function_definition">function body</DocLink>.

If an exception is thrown from its `compound-statement` or `ctor-initializer` (if any), the exception will be matched against the <DocLink dest="cpp/language/catch">handlers</DocLink> in its `handler-seq`:

```cpp
int f(bool cond)
{
    if (cond)
        throw 1;
    return 0;
}

struct X
{
    int mem;

    X() try : mem(f(true)) {}
    catch (...)
    {
        // handles the exception 1
    }

    X(int) try
    {
        throw 2;
    }
    catch (...)
    {
        // handles the exception 2
    }
};
```

Exceptions thrown in destructors of objects with static <DocLink dest="cpp/language/storage_duration">storage duration</DocLink> or in constructors of objects associated with <DocLink dest="cpp/language/scope#Block_scope">non-block variables</DocLink> with static storage duration are not caught by a function `try` block on the <DocLink dest="cpp/language/main_function">`main` function</DocLink>.

<RevisionBlock since="C++11">
Exceptions thrown in destructors of objects with thread storage duration or in constructors of objects associated with non-block variables with thread storage duration are not caught by a function `try` block on the initial function of the thread.
</RevisionBlock>

Flowing off the end of the `compound-statement` of a <DocLink dest="cpp/language/catch">handler</DocLink> of a function `try` block is equivalent to flowing off the end of the `compound-statement` of that function `try` block, unless the function is a constructor or destructor (see below).

#### Constructor and destructor `try` block

For a class `C`, if the function body of its constructor or destructor definition is a function `try` block, and an exception is thrown during the initialization or destruction, respectively, of `C`'s subobjects, the exception will also be matched against the <DocLink dest="cpp/language/catch">handlers</DocLink> in the `handler-seq` of the function `try` block:

```cpp
int f(bool cond = true)
{
    if (cond)
        throw 1;
    return 0;
}

struct X
{
    int mem = f();

    ~X()
    {
        throw 2;
    }
};

struct Y
{
    X mem;

    Y() try {}
    catch (...)
    {
        // handles the exception 1
    }

    ~Y() try {}
    catch (...)
    {
        // handles the exception 2
    }
};
```

Referring to any non-static member or base class of an object in the handler for a function `try` block of a constructor or destructor for that object results in undefined behavior.

If a <DocLink dest="cpp/language/return">`return` statement</DocLink> appears in a handler of the function `try` block of a constructor, the program is ill-formed.

The <DocLink dest="cpp/language/catch#Activating_the_handler">currently handled exception</DocLink> is rethrown if control reaches the end of a handler of the function `try` block of a constructor or destructor.

### Control flow

The `compound-statement` of a `try` block is a <DocLink dest="cpp/language/statements#Control-flow-limited_statements">control-flow-limited statement</DocLink>:

```cpp
void f()
{
    goto label;     // error
    try
    {
        goto label; // OK
        label: ;
    }
    catch (...)
    {
        goto label; // error
    }
}
```

A <DocLink dest="cpp/language/statements#Jump_statements">jump statement</DocLink> (<DocLink dest="cpp/language/goto">`goto`</DocLink>, <DocLink dest="cpp/language/break">`break`</DocLink>, <DocLink dest="cpp/language/return">`return`</DocLink>, <DocLink dest="cpp/language/continue">`continue`</DocLink>) can be used to transfer control out of a `try` block (including its handlers). When this happens, each variable declared in the `try` block will be destroyed in the context that directly contains its declaration:

```cpp
try
{
    T1 t1;
    try
    {
        T2 t2;
        goto label; // destroy t2 first, then t1
    }
    catch(...)
    {
        // executed if an exception is thrown while destroying t2
    }
}
catch(...)
{
    // executed if an exception is thrown while destroying t1
}
label: ;
```
